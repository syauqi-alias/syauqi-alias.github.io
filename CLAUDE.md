# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Install dependencies (first time or after Gemfile changes)
bundle install

# Local development server (live reload at http://localhost:4000)
bundle exec jekyll serve

# Production build (output in ./_site/)
bundle exec jekyll build
```

Deployment is automatic: pushing to `master` triggers the GitHub Actions workflow (`.github/workflows/jekyll.yml`), which builds and deploys to GitHub Pages at `https://syauqi-alias.github.io/`.

## Architecture

This is a Jekyll static site using the [`sighingnow/jekyll-gitbook`](https://github.com/sighingnow/jekyll-gitbook) remote theme, which gives it a GitBook-style sidebar-navigation layout.

**Content organisation:**
- `README.md` — the home/introduction page (`layout: home`)
- `_posts/` — main portfolio pages (`layout: post`); appear in the sidebar in **reverse** chronological order (newest date = first in sidebar). Date in the filename (`YYYY-MM-DD`) controls ordering, not actual publish date.
- `_pages/` — a custom Jekyll collection (`layout: post`); appears in the sidebar after posts in the order they appear on disk.

**Front matter required on every page:**
```yaml
---
title: Page Title
layout: post   # or "home" for README.md only
category: Jekyll
---
```

**Sidebar navigation** is auto-generated by `_includes/toc-date.html`, which iterates `site.posts` (reversed) then `site.pages` collections. In-page TOC headings are driven by `site.toc` settings in `_config.yml` (h1–h3 by default).

**Images:** Stored under `assets/`. Local images in `_posts` should use the full GitHub Pages URL (`https://syauqi-alias.github.io/assets/...`) rather than relative paths, because the `baseurl` is set to `/jekyll-gitbook` which affects relative resolution.

**Callout blocks** (from the gitbook theme):
```markdown
> ##### TIP
> Content here.
{: .block-tip }

> ##### WARNING
> Content here.
{: .block-warning }
```

**Markdown engine:** kramdown with GFM input, Rouge syntax highlighting (`colorful` style), and MathJax for equations.
